- **Target Users**: Coaches managing multiple teams per week

---

## 🎯 Why

When a coach manages multiple teams (1st XV, 2nd XV, etc.) in the same week, it's easy to accidentally double-book a player or miss someone who was already assigned to another team. Adding a small blue dot next to a player's name — both on the Availability Grid and in the Selection Board's Add Player pool — gives coaches an instant visual cue that a player is already selected somewhere, without needing to cross-reference teams manually.

---

## 🏗️ Architecture Decisions (Locked)

1. **No database changes needed**: `team_selections` already stores all teams' `player_order` arrays per week. The data is already there — we just need to compute a `Set<string>` of all assigned player IDs.
2. **`useSelectionBoard.ts` already fetches ALL team selections** for the active week (line 192-196). The `selections` state contains every team's selections. We just need to expose a computed `selectedPlayerIds` Set.
3. **`Grid.tsx` does NOT currently fetch `team_selections`** — it has its own inline data fetching. We must add a `team_selections` fetch for the current week.
4. **Inline styles only**: Both `Grid.tsx` and `SelectionBoard.tsx` use inline `style={}` objects exclusively (no Tailwind classes). The blue dot must use inline styles to match.
5. **Dot design**: `width: 8, height: 8, borderRadius: '50%', background: '#3B82F6'` (blue-500 equivalent), with `title` attribute for tooltip context.

## ⚠️ Edge Cases (Already Handled)

1. **No active week in Grid.tsx**: If `currentWeekId` is null, the `team_selections` fetch is skipped (`Promise.resolve({ data: [], error: null })`) and `selectedPlayerIds` remains an empty Set — no dots shown.
2. **No team selections exist**: `selRes.data` will be an empty array, the loop produces an empty Set — no dots shown. Correct behavior.
3. **Player is on multiple teams**: The Set deduplicates automatically — the dot appears once. Correct.
4. **`player_order` contains null values**: The `if (pid !== null)` guard in the Grid.tsx loop and the `.filter((id): id is string => id !== null)` in `useSelectionBoard.ts` both handle this.
5. **SelectionBoard reactivity**: The `selectedPlayerIds` is a `useMemo` dependent on `selections`. When `assignPlayer` or `removePlayer` updates `selections`, the Set recomputes and the dot appears/disappears immediately without a page refresh.
6. **Grid.tsx does NOT auto-refresh on selection changes**: The Grid fetches data once on mount via `fetchAll`. If a coach assigns a player after the Grid has loaded, they'd need to refresh the page to see the dot. This is acceptable — the Grid is a dashboard view, not a real-time tool. (The Selection Board is the real-time tool and does update instantly.)
