## 🏗️ Architecture Decisions (Locked)

### Data Layer
1. **No RPC:** Client-side data transformation only. Fetch players, weeks, responses separately → map in-memory
2. **Database Schema:**
   - `club_settings.default_teams`: `TEXT[]` (PostgreSQL array), nullable, default `NULL`
   - `weeks.notes`: `TEXT`, nullable, max 1000 characters
   - `week_teams.opponent`: `TEXT`, nullable, max 100 characters
3. **CSS Variables:** Override `--primary` (not create new `--color-brand`) for consistency with ARM-RULES.md
4. **Validation:** Client-side validation for hex colors, URLs, array lengths

### Performance
1. **No Virtualization:** 60 players × 10 weeks = 600 DOM nodes → standard React with `useMemo` sufficient
2. **`useMemo` for Matrix:** Memoize 2D availability dictionary transformation
3. **`React.memo` for Rows:** Prevent unnecessary re-renders of table rows
4. **Debounced Filtering:** If search added later

### CSS & Layout (Critical)
1. **Sticky Headers:** Three-layer z-index system for iOS Safari compatibility
2. **Intersection Cell:** First `<th>` in `<thead>` with highest z-index (not separate div)
3. **Background Colors Mandatory:** Prevents bleed-through on mobile WebKit
4. **Overlay Scroll Fixes:** `overscroll-behavior: contain`, lock body scroll when modals open
5. **Test on iOS Safari:** Mandatory acceptance criteria

### Fallbacks & Edge Cases
1. **Empty `default_teams`:** Fallback to `["1st XV", "2nd XV"]`
2. **Invalid Brand Color:** Show error, prevent save, suggest accessible alternatives
3. **Network Failures:** Preserve form data, show retry toast
4. **PWA Updates:** Update `theme_color` meta tag with brand color (defer `apple-mobile-web-app-title` to Phase 13)

### Critical Fixes (Based on Review)
1. **Color Contrast:** Dynamic text color based on brand color luminance (`getContrastColor()`)
2. **Logo Validation:** "Try to Load" approach instead of strict extension validation
3. **Schema Semantics:** Default teams are COPIED (not referenced) when creating weeks
4. **Database Constraints:** Add `CHECK (array_length(default_teams, 1) <= 10)`

## 📋 PART 2: Overlay Scroll Fixes ("Horizontal Jiggle" & "Scroll Chaining")

### Problem
Overlays/modals cause "horizontal jiggle" when scrolling and allow "scroll chaining" to background elements.

### Technical Context
- **Framework:** React with inline styles and some Tailwind
- **Overlay Structure:** Multiple components use `position: fixed`, `inset: 0`, `overflow-y: auto` patterns
- **Affected Components:** `PlayerOverlay.tsx`, `SelectionBoard.tsx`, `PlayerFormSheet.tsx`, `DeletePlayerDialog.tsx`, modal overlays in `Weeks.tsx` and `ResultDetail.tsx`

### Implementation Guidance
1. **Add to overlay containers:** `overscroll-behavior: contain`
2. **Lock body scroll when modals open:**
   ```typescript
   useEffect(() => {
     document.body.style.overflow = 'hidden';
     return () => {
       document.body.style.overflow = '';
     };
   }, []);
   ```
3. **Fix viewport calculations:** Use `100%` instead of `100vw` to avoid scrollbar issues
4. **Test on:** iOS Safari and Chrome Android

### `src/lib/colorUtils.ts` — NEW
```typescript
// Utility for dynamic text color based on background luminance
export function getContrastColor(hex: string): 'text-white' | 'text-gray-900' {
  // Remove # if present
  const cleanHex = hex.replace('#', '');
  
  // Convert to RGB
  const r = parseInt(cleanHex.substr(0, 2), 16);
  const g = parseInt(cleanHex.substr(2, 2), 16);
  const b = parseInt(cleanHex.substr(4, 2), 16);
  
  // Calculate relative luminance (WCAG formula)
  const luminance = (0.299 * r + 0.587 * g + 0.114 * b) / 255;
  
  // Return black text for light backgrounds, white for dark
  return luminance > 0.179 ? 'text-gray-900' : 'text-white';
}

// Validate hex color
export function isValidHexColor(hex: string): boolean {
  return /^#([A-Fa-f0-9]{6}|[A-Fa-f0-9]{3})$/.test(hex);
}
```
