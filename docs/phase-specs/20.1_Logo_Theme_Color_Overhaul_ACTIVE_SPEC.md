## 🎯 Why This Matters

The ARM app is getting a brand refresh. The legacy `Logo.png`/`Logo.PNG` files and Apple Touch icons need to be replaced with new assets. This spec covers:

1. **New asset placement** — User drops `app-icon.png` and `short-form-logo.png` into `public/icons/`
2. **HTML/PWA manifest updates** — Point all icon references to the new files
3. **UI component updates** — Replace `/icons/Logo.png` references in React components
4. **Color theme preparation** — Identify all hardcoded brand colors so the user can supply new hex codes

## 🏗️ Architecture Decisions

### 1. New Asset Convention
- `app-icon.png` → Used for Apple Touch Icon, PWA manifest icons, favicon
- `short-form-logo.png` → Used for UI logo rendering (Sidebar, Login, AvailabilityForm fallback, offline page)
- Both files placed in `/public/icons/` by the user

### 2. Logo Reference Strategy
- Replace all `/icons/Logo.png` → `/icons/short-form-logo.png`
- The `short-form-logo.png` is expected to be a compact/iconic version suitable for UI chrome (sidebar header, login card)
- The `app-icon.png` is expected to be a square app icon suitable for home screens

### 3. Color Strategy
- Colors are defined as CSS custom properties in `:root` in `src/index.css`
- Tailwind config is currently empty (`extend: {}`) — no custom colors defined
- All purple usage is either CSS variables or hardcoded Tailwind classes
- **Recommendation**: Add custom colors to `tailwind.config.js` so the app uses a single source of truth, then update the CSS variables to match

## ⚠️ Risks & Mitigations

### Risk 1: New assets have different aspect ratios
- **Mitigation**: The `short-form-logo.png` is rendered with `h-10 w-auto object-contain` in Sidebar and `maxHeight: '56px'` in Login — these should accommodate any reasonable aspect ratio
- **Fallback**: Adjust CSS sizing classes if the new logo looks disproportionate

### Risk 2: Case sensitivity on deployment (Vercel Linux)
- **Mitigation**: All new references use lowercase paths (`/icons/short-form-logo.png`)
- **Verification**: Test on production after deploy

### Risk 3: Service worker caches old logo
- **Mitigation**: Service worker will fetch new assets on next activation
- **Workaround**: Hard refresh (Ctrl+F5) clears the service worker cache
