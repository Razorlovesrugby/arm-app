## 🎯 Why This Matters
When users add the ARM app to their iPhone Home Screen, they expect:
- Consistent, high-quality icons that match the brand
- Professional splash screens during app launch  
- No broken images or placeholder icons
- A native-like PWA experience

Currently, we have inconsistent logo usage with both `Logo.PNG` (uppercase) and `Logo.png` (lowercase) files, and potential issues with iOS Home Screen icon display. This spec ensures all logo references are standardized and iOS is fully optimized.

## 🧠 Current State Analysis

### Existing Logo Files (`/public/icons/`):
- `Logo.PNG` - Primary source logo (uppercase)
- `Logo.png` - Duplicate? (lowercase)
- `apple-touch-icon.png` - iOS touch icon (may not be proper size)
- `icon-192.png` - PWA small icon
- `icon-512.png` - PWA large icon

### Code References Found:

2. **`public/manifest.json`** - References `/icons/icon-192.png` and `/icons/icon-512.png`

4. **`src/components/Sidebar.tsx`** - References `/icons/Logo.png`
5. **`src/pages/AvailabilityForm.tsx`** - References `/icons/Logo.png` (fallback logo)
6. **`public/offline.html`** - References `/icons/Logo.png`

### Issues Identified:
1. **Case inconsistency** - Both `Logo.PNG` and `Logo.png` exist
2. **iOS icon sizing** - `apple-touch-icon.png` may not be proper 180x180 size
3. **Missing splash screens** - No iOS startup images for different devices
4. **Manual process** - No automated icon generation pipeline

## 🏗️ Architecture Decisions

### 1. Single Source of Truth
- `Logo.PNG` will be the canonical source logo file
- All derivative icons will be generated from this source
- Maintain original in `/public/icons/Logo.PNG`

### 2. Case Consistency Strategy
- Create lowercase `logo.png` as compatibility copy
- All code references should use lowercase `/icons/Logo.png`
- Case-sensitive systems will work with lowercase version

### 3. iOS Optimization Standards
- iOS touch icon: 180x180 pixels (iPhone retina)
- iPad icons: 167x167 (Pro), 152x152 (regular)
- Splash screens: Device-specific with brand color background
- Meta tags: Complete iOS PWA configuration in `index.html`

### 4. Icon Generation Pipeline
- Create simple script to generate all icons from source
- Document process for future logo updates
- Ensure all icons maintain visual consistency

## ⚠️ Risks & Mitigations

### Risk 1: Logo.PNG has insufficient resolution
- **Mitigation**: Check dimensions first, recommend minimum 512x512
- **Fallback**: Use existing `icon-512.png` as source if needed

### Risk 2: Case sensitivity issues on deployment
- **Mitigation**: Ensure both `Logo.PNG` and `logo.png` exist
- **Fallback**: Use lowercase references everywhere

### Risk 3: iOS splash screens not displaying
- **Mitigation**: Test on actual iOS devices
- **Fallback**: At minimum ensure icon works, splash is enhancement

### Risk 4: Club custom logos broken
- **Mitigation**: Preserve `logo_url` functionality in Club Settings
- **Fallback**: Default to `/icons/Logo.png` if custom fails
