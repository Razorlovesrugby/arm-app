### Modify `createWeek` - **DYNAMIC TEAMS**
Remove `DEFAULT_TEAM_NAMES`. Update interface and implementation:

```typescript
export interface CreateWeekParams {
  start_date: string
  end_date: string
  label: string
  teamNames: string[]  // NEW: Array of custom team names
}

const createWeek = useCallback(async ({ start_date, end_date, label, teamNames }: CreateWeekParams) => {
  // Validation: minimum 1 team, no duplicates (case-insensitive, trimmed)
  if (teamNames.length === 0) {
    return { data: null, error: 'At least one team is required' }
  }
  
  const trimmedNames = teamNames.map(name => name.trim())
  const uniqueNames = new Set(trimmedNames.map(name => name.toLowerCase()))
  if (uniqueNames.size !== trimmedNames.length) {
    return { data: null, error: 'Duplicate team names are not allowed' }
  }

  // Create week with token
  const token = crypto.randomUUID()
  const { data: weekData, error: weekError } = await supabase
    .from('weeks')
    .insert({ start_date, end_date, label, status: 'Open', availability_link_token: token })
    .select()
    .single()

  if (weekError || !weekData) {
    return { data: null, error: weekError?.message ?? 'Failed to create week' }
  }

  // Insert custom team rows
  const teamRows = trimmedNames.map((name, i) => ({
    week_id: weekData.id,
    team_name: name,
    sort_order: i + 1,
    starters_count: 15,
    is_active: true,
    visible: true
  }))

  const { error: teamsError } = await supabase
    .from('week_teams')
    .insert(teamRows)

  if (teamsError) {
    console.error('Failed to insert week_teams:', teamsError)
    return { data: weekData, error: `Week created but teams failed: ${teamsError.message}` }
  }

  await fetchWeeks()
  return { data: weekData, error: null }
}, [fetchWeeks])
```

```tsx
const months = useMemo(() => {
  const monthSet = new Set<string>()
  openWeeks.forEach(week => {
    const date = new Date(week.start_date + 'T00:00:00Z') // UTC for consistency
    const monthKey = date.toLocaleDateString('en-US', { month: 'short' }).toUpperCase()
    const year = date.getFullYear()
    // Include year if multiple years present
    const hasMultipleYears = new Set(openWeeks.map(w => new Date(w.start_date + 'T00:00:00Z').getFullYear())).size > 1
    const label = hasMultipleYears ? `${monthKey} '${year.toString().slice(-2)}` : monthKey
    monthSet.add(label)
  })
  return Array.from(monthSet).sort((a, b) => {
    // Simple alphabetical sort works for months within same year
    // For cross-year, would need custom logic
    return a.localeCompare(b)
  })
```

## 4. UI: Create Week Flow - Dynamic Teams (`CreateWeekForm`)

### Teams Playing Section - **DYNAMIC INPUTS**
```tsx
const [teams, setTeams] = useState<{ id: number; name: string }[]>([
  { id: 1, name: '1st XV' }
])

const addTeam = () => {
  setTeams([...teams, { id: Date.now(), name: '' }])
}

const removeTeam = (id: number) => {
  if (teams.length <= 1) return // Minimum 1 team
  setTeams(teams.filter(team => team.id !== id))
}

const updateTeam = (id: number, name: string) => {
  setTeams(teams.map(team => 
    team.id === id ? { ...team, name } : team
  ))
}

// In validation before submit:
const teamNames = teams.map(t => t.name.trim()).filter(name => name !== '')
if (teamNames.length === 0) {
  setErrors(prev => ({ ...prev, teams: 'At least one team name is required' }))
  return false
}

// Check for duplicates (case-insensitive)
const lowerNames = teamNames.map(name => name.toLowerCase())
const uniqueNames = new Set(lowerNames)
if (uniqueNames.size !== teamNames.length) {
  setErrors(prev => ({ ...prev, teams: 'Duplicate team names are not allowed' }))
  return false
}
```

## 5. Edge Cases & Rules

### Month Filtering
- **Timezone:** Use UTC date parsing: `new Date(start_date + 'T00:00:00Z')`
- **Month Determination:** Use `start_date` only (even if week spans two months)
- **Empty Months:** Don't show month pills with zero open weeks
- **Multiple Years:** Show "JAN '26" vs "JAN '27" when years differ

### Label Syncing
- `updateWeek` updates database → triggers `fetchWeeks` → updates all components
- `AvailabilityForm.tsx` uses `useWeeks` hook, so it gets fresh data automatically
- **No cache invalidation needed** - hook provides single source of truth

### Team Validation
- **Duplicate Teams:** Case-insensitive check, trim whitespace before comparison
- **Empty Names:** Teams with only whitespace are treated as empty and removed before validation
- **Minimum:** At least 1 non-empty team name required

### Error States & Loading
- **Inline Edit:** Show saving spinner, error toast on failure, revert on cancel
- **Month Filter:** Show skeleton loading when filtering (if async needed)
- **Team Inputs:** Real-time validation feedback (red border on duplicates)

### What NOT to Change
- Do not alter `/results` or `match_events` logic (Phase 12.2)
- Do not touch Selection Board logic (`/board`)
- Do not change WhatsApp Share message generation (except it uses updated `week.label`)
