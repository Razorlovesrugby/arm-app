## Part 2: Always Visible Results Ledger

### ✅ Implementation Status

| Feature | Status | Notes |
|---------|--------|-------|
| Show All Weeks | ✅ Done | Results tab shows past, present, and future weeks |
| Upcoming Badge | ✅ Done | Weeks with no scores show "Upcoming" badge |
| Safe Null Score Display | ✅ Done | Empty scores show "– : –" instead of "0 : 0" |
| Fixture & Results View | ✅ Done | Comprehensive ledger of all scheduled/completed games |
| Automatic Inclusion | ✅ Done | Open weeks automatically added to results (no filtering needed) |

### 🔧 Technical Implementation

#### 1. Results Page Transformation (`src/pages/Results.tsx`)
- **Removed Date Filtering:** Previously filtered to `pastWeeks` only, now shows ALL weeks
- **Upcoming Logic:** `isUpcoming = activeTeams.every(t => t.score_for === null)`
- **Badge Display:** "Upcoming" badge shown in gray (`bg-gray-100 text-gray-500`)
- **Score Display:** Safe null handling: `team.score_for !== null ? \`${team.score_for} – ${team.score_against ?? 0}\` : '– : –'`

#### 2. Data Flow
- **Hook Usage:** Uses `useWeeks()` which returns all weeks (no filtering)
- **Week Ordering:** Maintains descending `start_date` order
- **Active Teams Filter:** Only shows teams where `is_active !== false`
- **Match Report Preview:** Shows first 50 characters of match report if available

### 📋 Acceptance Criteria Met

- [x] Results tab shows ALL weeks (past, present, future) — not just past weeks
- [x] "Upcoming" badge displayed for weeks with no scores entered
- [x] Safe null score display ("– : –") instead of "0 : 0"
- [x] Clickable cards navigate to `/results/:weekId` detail view
- [x] Match report preview shown (first 50 characters)
- [x] Active teams only displayed (filters out `is_active=false` teams)
- [x] Maintains chronological order (most recent first)

## Performance Notes

1. **Month Derivation:** Memoized with `useMemo` based on `openWeeks`
2. **Availability Counts:** Pre-calculated in hook, not in render
3. **Filtered Weeks:** Memoized based on `openWeeks` and `selectedMonth`
4. **Network Efficiency:** Two separate efficient queries instead of one bloated nested query
5. **Icon Imports:** Tree-shaken from lucide-react
6. **State Management:** Optimistic updates for label editing with rollback on error
