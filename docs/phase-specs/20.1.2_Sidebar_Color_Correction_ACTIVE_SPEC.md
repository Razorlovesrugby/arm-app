## 🎯 Why This Matters

The Sidebar currently uses `bg-brand-dark` (`#7B2FFF` — ultraviolet) as its background. The user wants the Sidebar to use the **primary blue** (`#0062F4`) instead, with navigation link hover states using the **light blue** (`#E8F0FE`) background with dark text for contrast.

---

## 🎨 Current vs. Target

| Element | Current | Target |
|---------|---------|--------|
| Sidebar background | `bg-brand-dark` (`#7B2FFF`) | `bg-brand` (`#0062F4`) |
| Nav link hover | `hover:bg-brand` (`#0062F4`) | `hover:bg-[#E8F0FE]` with `hover:text-[#0062F4]` |
| Nav link active | `bg-brand` (`#0062F4`) | Keep as-is (or adjust for contrast) |
| Nav link default text | `text-brand-light` (`#E8F0FE`) | Keep as-is |
| Header border | `border-brand` (`#0062F4`) | Keep as-is (or adjust) |
| Close button hover | `hover:bg-brand` (`#0062F4`) | Keep as-is |
| Sign out button | `bg-brand hover:bg-brand-dark` | Keep as-is |

## ⚠️ Risks & Mitigations

### Risk 1: Active nav link blends into sidebar
- The active link uses `bg-brand border-l-4 border-white font-bold` — since the sidebar is now also `bg-brand`, the active state won't be visually distinct.
- **Mitigation**: Change active link background to `bg-white/15` so it's subtly highlighted against the `bg-brand` sidebar.

### Risk 2: `hover:bg-[#E8F0FE]` is an arbitrary value
- This works in Tailwind v3 but won't be caught by a purge step.
- **Mitigation**: If this pattern is used elsewhere, consider adding a `brand-hover-bg` token to the Tailwind config.
