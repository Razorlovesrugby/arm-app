```sql
-- 2. Create rdo_club_access bridging table
CREATE TABLE IF NOT EXISTS rdo_club_access (
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,

  PRIMARY KEY (user_id, club_id)

-- 3. Performance index for EXISTS subqueries
CREATE INDEX IF NOT EXISTS idx_rdo_club_access_user_club 
  ON rdo_club_access(user_id, club_id);

-- 4. Expand RLS policies on all 10 core tables with OR logic
-- (Example for players table, repeated for all 10 tables)
DROP POLICY IF EXISTS players_coach_all ON players;
CREATE POLICY players_coach_all ON players
  FOR ALL TO authenticated
  USING (
    -- Original coach logic
    club_id IN (SELECT club_id FROM profiles WHERE id = auth.uid())
    OR
    -- New RDO logic
    EXISTS (
      SELECT 1 FROM rdo_club_access 
      WHERE user_id = auth.uid() AND club_id = players.club_id
    )
  )
  WITH CHECK (same logic);
```

9. `training_attendance` - Training attendance records
10. `archive_game_notes` - Archived match notes

### RLS Policy Pattern
Each policy follows this pattern:
```sql
USING (
  -- Coach access: user's profile club_id matches row's club_id
  club_id IN (SELECT club_id FROM profiles WHERE id = auth.uid())
  OR
  -- RDO access: user is mapped to row's club_id via rdo_club_access
  EXISTS (
    SELECT 1 FROM rdo_club_access 
    WHERE user_id = auth.uid() AND club_id = table_name.club_id
  )
)
```

### Performance Optimizations
1. **Composite Index**: `idx_rdo_club_access_user_club` ensures fast EXISTS subqueries
2. **Idempotent Migration**: All operations use `IF NOT EXISTS`/`IF EXISTS` checks
3. **Backward Compatibility**: Existing coach workflows unchanged

## 🎨 Architecture Decisions

### 1. Role-Based Access Control
- **Two-Tier System**: `'coach'` (single club) vs `'rdo'` (multi-club)
- **Backward Compatible**: Existing users remain `'coach'` by default
- **Constraint Enforcement**: Database-level CHECK constraint prevents invalid roles

### 2. Bridging Table Design
- **Composite Primary Key**: `(user_id, club_id)` ensures unique mappings
- **CASCADE Delete**: Automatically cleans up mappings when users or clubs deleted
- **Audit Trail**: `created_at` timestamp for tracking access grants

### 3. RLS Expansion Strategy
- **OR Logic**: Policies expanded rather than replaced to avoid conflicts
- **Service Role Preservation**: Admin tooling bypass remains intact
- **Anonymous Policies**: Public availability form policies unchanged
